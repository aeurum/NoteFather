NoteFather ExportData
The NoteFather is a bot that allows to create reminders, notes, and task lists without leaving Telegram. In order to export all of your NoteFather data, send the command /export to the bot and download the file you shall receive in seconds. The file will contain a JSON‑serialized object consisting of 2 arrays that comprise your contents.
Item
This object represents an element of the “items” array.
Field
Type
Description
chat_id
Integer
Your Telegram identifier
id
Integer
Unique item identifier
type
String
Type of the item; can be either “reminder” or “note”
alert
Integer or NULL
Date of the notification in Unix time; equal to NULL if type is “note”
priority
Integer
Item priority; 1 for huge, 2 for high, 3 for medium, and 4 for low
data_type
String
One of the available Telegram data types
text
String or NULL
Text of the item, 0–4096 characters
file_id
String or NULL
Unique file identifier; equal to NULL if date_type is “text”, “location”, “venue”, or “contact”.
is_list
Boolean
True if the item contains a list
can_others_add
Boolean
True if anyone having access to the list may add new tasks thereto
is_repeat
Boolean
True if the reminder is repeating
repeat_number
JSON
Represents a way the repeats shall stop; can be either a JSON‑serialized Boolean, String, or Integer:

1. if Boolean, repeats shall never stop;

2. if String, it represents the date in Unix time (converted to a string) when repeats shall stop;

3. if Integer, it represents a specific number of remaining repeats.
period_type
String or NULL
Type of the repeat period; can be either “y”, “m”, “f”, “w”, “d”, “h”, or “i”, which stand for “years”, “months”, “fortnights”, “weeks”, “days”, “hours”, and “minutes” respectively
period_range
JSON
Represents an interval between repeats; can be either a JSON‑serialized Integer, Array, or String:

1. if Integer, it represents a number of periods that shall pass between every 2 repeats; e.g. if period_type is “d” and deserialized period_range is equal to 180, the reminder shall repeat every 180 days;

2. if Array, it represents a set of days or months when the reminder shall repeat; e.g. if period_type is “d” and deserialized period_range is like [1, 3, 5], the reminder shall repeat every Monday, Wednesday, and Friday;

3. if String, it can be either “e” or “o”, which stand for “even” and “odd”, so the reminder shall repeat every respective day.
is_paused
Boolean
True if repeats are currently paused
paused_until
Integer or NULL
Date repeats shall be unpaused in Unix time; if NULL, repeats may be unpaused only manually by the user
is_sent
Boolean
True if the reminder is completed (not scheduled for a date in the future)
date
Integer
Date the reminder was created in Unix time
